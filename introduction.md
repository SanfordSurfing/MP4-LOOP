# 项目文件速览（口语化说明）

- `README.md`：项目说明文档，包含快速开始指南、技术栈说明、功能特性、项目结构等。现在使用后端架构（Node.js + Express + FFmpeg），完全避免 CORS 问题。
- `package.json`：Node.js 项目的依赖配置文件。定义了项目需要的 npm 包（express、multer、fluent-ffmpeg、cors）和启动脚本。
- `server.js`：**后端服务器**，这是核心处理逻辑：
  - 使用 Express 框架创建 Web 服务器
  - 接收前端上传的视频文件
  - 使用 FFmpeg 处理视频循环（1/3/5 次）
  - 支持 MP4 和 GIF 两种输出格式
  - 处理完成后返回文件给前端下载
  - 自动清理临时文件
- `index.html`：前端页面骨架，包含上传区、循环与质量选择、格式选择、进度与预览、下载按钮，以及简易法务说明。默认英文文案，预留多语言入口。
- `styles.css`：Apple 风格的深色玻璃拟态样式，响应式布局，按钮/卡片/进度条/标签等基础样式。
- `app.js`：**前端逻辑**，现在只负责：
  - 用户界面交互（上传、选择、按钮点击）
  - 调用后端 API（`/api/process`）上传视频并获取处理结果
  - 显示进度和预览
  - 下载处理后的文件
  - 不再包含任何视频处理逻辑，所有处理都在后端完成
- `vendor/`：之前存放 gif.js 的目录，现在不再使用（后端直接使用 FFmpeg 生成 GIF）。

**技术架构**：
- **前端**：纯静态页面，只负责 UI 和 API 调用
- **后端**：Node.js + Express + FFmpeg，所有视频处理都在服务器端完成
- **优势**：完全避免 CORS 问题，处理速度更快，支持更多视频格式

**使用流程**：
1. 启动后端服务器：`npm start`
2. 浏览器访问：`http://localhost:3000`
3. 上传视频 → 后端处理 → 下载结果
